# P5 历史能力降级与叙事收口完成报告

> 评估时间：2026-04-02
> 评估范围：`mss-boot-admin` 三期路线图 P5 阶段

## 1. P5 定义回顾

### 目标

> 控制历史能力的对外叙事与后续投资预期，使产品主线不再被旧叙事反复打断。

### 覆盖范围

- 动态模型
- 模板/代码生成
- 历史"开发辅助能力"的对外表述

### 核心问题

1. 哪些历史能力继续保留为存量能力？
2. 哪些能力进入冻结或弱化状态？
3. 文档中如何表达"仍存在"与"非主线"的关系？

### 预期结果

1. 产品主线不再被旧叙事反复打断
2. 团队和外部使用者都能理解当前重点到底是什么

## 3. 分级方案

### 分级定义

| 级别 | 名称 | 含义 | 后续投资 |
|------|------|------|----------|
| L1 | 保留 | 继续作为核心能力 | 正常投入 |
| L2 | 冻结 | 保留功能，不再新增 | 仅修复严重问题 |
| L3 | 弱化 | 标记为非主线 | 最小维护 |
| L4 | 弃用 | 计划移除 | 仅安全修复 |
| L5 | 删除 | 已移除 | 无 |

### 历史能力分级

| 能力 | 分级 | 保留原因 | 限制说明 |
|------|------|----------|----------|
| 动态模型 | L3 弱化 | 部分用户可能依赖 | 不作为主线推荐，文档标注为"高级功能" |
| 代码生成 | L3 弱化 | 可作为扩展机制 | 无内置模板，用户需自行准备 |

**正确表述：**
```
✅ "mss-boot-admin 是一个治理型 + 运营型 admin 平台"
✅ "动态模型能力仍保留，适合快速原型开发，但不作为主线推荐"
✅ "生产环境建议使用标准 Controller 模式开发"
```

```
❌ "mss-boot-admin 是一个低代码平台"
❌ "动态模型是 mss-boot-admin 的核心能力之一"
❌ "支持快速生成 CRUD 代码"
```
